# One User Rule  
**One user. Three metrics. No excuses.**

This is not a PRD generator.  
This is a product thinking discipline engine.

If your thinking depends on vibes, this will reject you.

---

## What This Is

**One User Rule** evaluates early PM thinking across:

- Problem clarity  
- Persona precision  
- Solution discipline  
- Metric integrity  
- Scope awareness  
- Ambition level  

It enforces constraints most PMs avoid.

You don’t get multiple personas.  
You don’t get five metrics.  
You don’t get safe ambition.

---

## The Rule

Every submission must follow:

- Exactly **ONE** core user  
- Maximum **THREE** measurable metrics  
- Every metric must include:
  - Numeric baseline  
  - Numeric target  
- PRD must not exceed **300 words**
- No vague modifiers:
  - “improve”
  - “optimize”
  - “intuitive”
  - “seamless”
  - “engagement” (without definition)

Break structure → instant rejection.

---

## Why This Exists

Common early PM mistakes:

- Multiple personas to avoid trade-offs  
- Metrics without baselines  
- 3–5% “safe” targets  
- Long PRDs with no measurable friction  
- Feature thinking disguised as strategy  

If you cannot pick one user, you do not understand the problem.  
If you cannot quantify pain, you do not understand the outcome.

---

## Maturity Levels

**Level 1 – Idea Thinker**  
Describes goals. No causality. Weak metrics.

**Level 2 – Feature Thinker**  
Defines a feature. Weak outcome logic.

**Level 3 – Metric-Aware PM**  
Has numbers. Lacks ambition or precision.

**Level 4 – Outcome-Oriented PM**  
Clear problem → solution → measurable impact.

**Level 5 – Strategic Operator**  
Tight persona. Strong causal chain. Bold but realistic ambition. Explicit trade-offs.

Scores above 8 are rare.  
Most submissions land between 5–7.

Below 5 means vague framing, disconnected solutions, or vanity metrics.
